reading guide

如何看懂这里

基本概念

Run
agent 的一次执行 —— 从接到一条用户输入，到它停止调用工具或结束。
Turn
一轮 LLM 决策：模型生成一段回复 + 可选的一批 tool 调用。一个 run 常见多个 turn。
Lane
事件分类，用颜色区分。

Lane 速查

  • dispatch入口事件：消息入队、注入到活跃 run。
  • runtimeagent 生命周期：started / turn / inbox / finished / failed。
  • llmLLM 请求与响应（成对展示为 llm.exchange）。
  • tool工具调用（成对展示为 tool.<name>）。
  • scheduler异步调度（Hermes / planner）。
  • memory记忆压缩的开始与结束。
  • artifact落盘产物：plan / diary / memory / history。

怎么用

  1. 列表页点开一条 run，进入它的 prompt 路径。
  2. 详情页先读顶上那段话，大致知道这个 run 发生了什么。
  3. Timeline 按 turn 分组 —— 左列是 request（发给模型的最后消息），右列是 response（模型输出 / 调用的 tool）。
  4. 底部 payload 折叠区可看原始 JSON。
